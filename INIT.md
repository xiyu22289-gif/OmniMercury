# INIT.md - 项目初始化文档

## 一、项目目标 (Project Goal)
以 **Mercury** 为参考原型，开发一款**跨平台桌面端、本地优先、支持通用大模型接入**的 RSS 阅读器。

## 二、团队分工 (Team Roles)

| 成员 | 角色定位 | 核心职责 |
| :--- | :--- | :--- |
| **成员 A** | **前端交互负责人** | 跨端框架搭建、界面布局、阅读渲染、主题样式、交互逻辑 |
| **成员 B** | **业务核心负责人** | 订阅源解析、内容清洗流水线、本地数据持久化、业务逻辑封装 |
| **成员 C** | **AI与工程化负责人** | LLM接入层、AI Agent实现、工程化构建、文档沉淀、协同管理 |

## 三、核心功能规划 (Core Features)

### 1. 必做核心功能 (Milestones 1-4)
*   **M1: 项目脚手架与 UI 骨架**
    *   内容：初始化工程、三栏布局框架、基础路由、本地数据库核心表结构、项目规范落地。
*   **M2: 订阅源管理与文章列表**
    *   内容：RSS/Atom/JSON Feed 解析、OPML 导入导出、同步刷新、文章列表渲染、已读/未读/星标状态、无限滚动。
*   **M3: 内容清洗流水线与阅读模式**
    *   内容：网页正文提取、广告/导航清洗、HTML 转 Markdown、Reader 纯净模式、原始网页模式切换、主题/字体定制。
*   **M4: LLM 通用接入与 AI 摘要/翻译**
    *   内容：LLM Provider 抽象层、服务商管理、流式调用封装、摘要 Agent、分段翻译 Agent、双语对照渲染、结果本地持久化。

### 2. 选做加分功能 (Milestones 5-8)
*   **M5: 标签系统**：手动打标签、AI 标签推荐、按标签筛选、标签库管理。
*   **M6: 笔记与文摘导出**：单篇笔记、单篇/多篇导出 Markdown 文摘（含摘要/笔记）。
*   **M7: 大模型用量统计**：Token 消耗记录、多维度统计报表（按服务商/模型/功能）。
*   **M8: 界面多语言**：中英双语界面、无缝切换。

### 3. 额外亮点扩展 (Optional)
*   全文搜索 (SQLite FTS5)、全键盘快捷键、自动同步与后台刷新、数据一键备份与恢复、系统主题自适应。

## 四、技术选型 (Technology Stack)

| 技术领域 | 选型方案 |
| :--- | :--- |

## 五、项目约束 (Constraints)
1.  **本地优先**：数据必须本地存储（SQLite），无需服务端，保障隐私。
2.  **大模型中立**：必须兼容 OpenAI 格式标准 API，支持本地/云端模型。
3.  **跨平台**：必须天然支持 Windows/macOS/Linux 三端运行。
4.  **禁止自研**：正文提取等核心难点必须使用成熟的第三方库（如 `@mozilla/readability`），禁止重复造轮子。
